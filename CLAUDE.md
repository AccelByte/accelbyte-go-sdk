# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Generated Go SDK (`github.com/AccelByte/accelbyte-go-sdk`) for AccelByte Gaming Services (AGS/"Justice"). Almost all code here is **generated** from OpenAPI specs by the sibling `justice-codegen-sdk` repo ‚Äî see the workspace-level `CLAUDE.md` one directory up for the full cross-repo pipeline (spec repo ‚Üí codegen tool ‚Üí this repo, tested against the mock-server repo). The exact codegen commit this repo was generated with is pinned in `codegen.txt`.

**Before hand-editing generated code, stop and check upstream.** Files under `*-sdk/pkg/*client/`, `*-sdk/pkg/*clientmodels/`, and most of `services-api/pkg/service/<service>/` start with `// Code generated. DO NOT EDIT.` ‚Äî a fix there almost always belongs in the spec's `.jsonpatch` files, a `modify-swagger.yml`/rename rule, or a generator template in `justice-codegen-sdk`, not as a local edit that will be silently overwritten on the next regen.

## Repo layout

- **`<service>-sdk/pkg/<service>client/`** ‚Äî low-level generated client (go-swagger style): one file per operation (`Parameters`, `Responses`) plus a `JusticeXxxService` client struct. Regenerated by `Makefile.sdk-go`'s `sdk_operations` target in `justice-codegen-sdk`.
- **`<service>-sdk/pkg/<service>clientmodels/`** ‚Äî generated request/response model structs (`sdk_models` target).
- **`services-api/pkg/service/<service>/`** ‚Äî higher-level `XxxService` wrapper structs (e.g. `iam.OAuth20Service`) that consumers actually instantiate. Each wrapper method wraps a low-level client call: attaches the bearer token from `TokenRepository`, unpacks the go-swagger multi-return (`ok, unauthorized, forbidden, notFound, err`) into a single `(payload, error)`. Most methods have two forms: a legacy one deprecated in favor of a `...Short` variant (same call, but takes an `authInfoWriter` instead of re-resolving the token repository each time ‚Äî see `oAuth20.go`). Regenerated by `sdk_wrappers`.
  - A few files here are genuinely hand-maintained (no generated-code header): `services-api/pkg/service/iam/auth.go`, `jwt.go`, `auth_validator.go` ‚Äî local/offline JWT token validation and permission-checking logic. This is where recent work has focused (see `12c24d56f`, `721619076`, `7a8e24638` ‚Äî cache-key correctness after IAM key rotation / role overrides).
- **`services-api/pkg/factory/`** ‚Äî generated `NewXxxClient(configRepository)` constructors that build a `JusticeXxxService` client from a base URL.
- **`services-api/pkg/repository/`** ‚Äî hand-maintained core interfaces every service wrapper depends on: `ConfigRepository` (client id/secret/base URL), `TokenRepository` (access/refresh token storage), `RefreshTokenRepository` (auto-refresh policy, `RefreshRate`/`AutoRefresh`).
- **`services-api/pkg/utils/`** ‚Äî shared runtime: `auth/` (login/logout, `Session`, `RefreshTokenScheduler`), `auth/validator/` (local token validation implementation), `connectionutils/` (websocket `ConnectionManager`, used for lobby/chat), `parser/`, retry policy, flight-ID (`X-Flight-Id`) handling, user-agent string.
- **`services-api/pkg/tests/sdk/`** ‚Äî unit tests for the runtime utils (http retry, request/response helpers, refresh token) ‚Äî no external services needed, part of `test_core`.
- **`services-api/pkg/tests/integration/`** ‚Äî one `_test.go` per service, hits a real AGS environment; needs `ENV_FILE_PATH` credentials, run via `test_integration`.
- **`samples/`** ‚Äî standalone Go programs demonstrating SDK usage, several with their own `go.mod` (so they're intentionally outside the main module and not linted/built by the top-level `go build ./...`): `cli` (backs `test_cli`, see `samples/cli/cmd` for command wiring and `samples/cli/pkg/repository` for a concrete `ConfigRepository`/`TokenRepository` implementation), `getting-started`, `custom-operation`, `oidc-web`, `aws-lambda-example`, `tic-tac-toe`, `ws-client-refresh-token`.
- **`docs/operations/`** ‚Äî generated per-endpoint usage docs; `docs/samples/` ‚Äî sample app docs.
- **`spec/`** ‚Äî the per-service OpenAPI spec JSON files this SDK was generated from (copied in from the spec pipeline; not hand-edited here).

## Commands

Everything below runs dockerized ‚Äî no local Go toolchain assumed, but Docker must be running.

```sh
make lint                                            # golangci-lint over every go.mod dir (main module + each samples/*/go.mod)
make lint-mod-tidy                                   # go mod tidy check, per module
make samples                                         # go mod tidy && go build for every samples/*/main.go
make test_core SDK_MOCK_SERVER_PATH=/path/to/justice-codegen-sdk-mock-server   # unit + service tests, needs mock-server + ws-mock-server + httpbin (started automatically)
make test_integration ENV_FILE_PATH=/path/to/.env    # tests against a real AGS environment
make test_cli SDK_MOCK_SERVER_PATH=/path/to/justice-codegen-sdk-mock-server    # builds samples/cli and runs its shell-script test suite (samples/cli/tests/*.sh) against the mock server
```

- `test_core` runs exactly these packages: `services-api/pkg/utils/...`, `services-api/pkg/model/...`, `services-api/pkg/repository/...`, `services-api/pkg/service`, `services-api/pkg/tests/sdk`. To run a single test, replicate the `docker run ... go test` invocation from the `test_core` target in the `Makefile` but scope it to `-run <Pattern>` and the specific package, e.g.:
  ```sh
  docker run --rm --tty --network host --user $(id -u):$(id -g) \
    --volume $(pwd):/data --workdir /data --env GOCACHE=/data/.cache/go-build \
    --env AB_HTTPBIN_URL=http://localhost golang:1.24.1 \
    sh -c "go test -v -race -run TestName github.com/AccelByte/accelbyte-go-sdk/services-api/pkg/tests/sdk"
  ```
  Note the mock-server/ws-mock-server/httpbin containers still need to be up first (see the `test_core` target) if the test exercises HTTP calls.
- `test_integration` needs real AGS credentials in the env file (`AB_BASE_URL`, `AB_CLIENT_ID`, `AB_CLIENT_SECRET`, etc.) ‚Äî never invent or hardcode these.
- golangci-lint config (`.golangci.yml`) disables a long, explicitly-annotated list of linters (each with a `# TODO: enable in ...` comment) ‚Äî check there before assuming a lint rule applies everywhere in the repo.
- Version bumps go through `make version` (bumps `version.txt`, `UserAgentSDK` string, and the `getting-started` sample's pinned version) ‚Äî not a manual edit.

## Conventions

- Copyright/license header + `// Code generated. DO NOT EDIT.` marks a file as generated; absence of the second line (but presence of the copyright header) marks intentionally hand-maintained files ‚Äî both are common even within the same package (e.g. `services-api/pkg/service/iam/`), so check the header before editing, not just the directory.
- Conventional Commits, enforced by commitlint (`commitlint.config.js`, checked in CI per the workspace root `CLAUDE.md`).
- Deprecation comments follow `// Deprecated: YYYY-MM-DD - <reason/replacement>`; `make outstanding_deprecation` flags any such comment older than 6 months as needing removal.
